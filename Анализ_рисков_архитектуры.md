# Анализ рисков архитектуры и компромиссов

| Риск | Описание | Способ снижения |
|------|----------|----------------|
| **Высокая сложность системы** | Множество сервисов, очередей, хранилищ | Документация, автоматизация CI/CD, стандарты разработки |
| **Много точек отказа** | Kafka, базы данных, API Gateway | Кластеризация, репликация, резервирование |
| **Рост операционных затрат** | Kubernetes + брокер сообщений + мониторинг | Автоматическое масштабирование и управление ресурсами |
| **Сложность отладки** | Распределённые сервисы и события | Распределённые трассировки (OpenTelemetry) |
| **Сложность синхронизации данных между сервисами** | Eventual consistency | Строгие схемы и контракты событий |
| **Возможные задержки** | Event-driven модель увеличивает латентность | Гибридная модель: REST + события |
| **Требование квалифицированной команды** | Микросервисная парадигма сложнее монолита | Обучение и стандарты разработки |

**Компромисс:** повышенная сложность инфраструктуры взамен на гибкость, масштабируемость и устойчивость системы.
