# Стоимость владения системой (TCO)

---

# 1. Первый год (MVP)

## CAPEX (разовые затраты)
- Развёртывание инфраструктуры, CI/CD: **$3,000–7,000**
- Первоначальная разработка MVP (платформа + ядро фич): **$20,000–40,000**
- Настройка мониторинга, логирования, безопасности: **$5,000–10,000**

**Итого CAPEX первого года:** **$28,000–57,000**

## OPEX (ежемесячные)
- Kubernetes-кластер: $300–600  
- PostgreSQL (1 instance): $200–400  
- Kafka (1 брокер): $150–300  
- S3/объектное хранилище: $50–150  
- Логи/мониторинг: $30–80  
- API Gateway + баланcировка: $50–150  
- Поддержка и сопровождение: $500–1,000  

**Итого OPEX:** ~$1,200–2,500 / месяц  
**Итого за год:** **$14,000–30,000**

---

# 2. Второй год (масштабирование и рост)

## CAPEX (разовые)
- Реализация новых сервисов (социальные фичи, рекомендации): **$10,000–20,000**
- Расширение архитектуры (миграция некоторых модулей в микросервисы): **$5,000–10,000**
- Стресс-тестирование и оптимизация: **$2,000–5,000**

**Итого CAPEX второго года:** **$17,000–35,000**

## OPEX (ежемесячные)
- Kubernetes (увеличение нод): $800–1,200  
- PostgreSQL high-availability: $400–700  
- Kafka (2–3 брокера): $400–700  
- S3 (до 2 ТБ): $100–300  
- Мониторинг/логи: $100–200  
- Поддержка DevOps/Сопровождение: $1,500–2,000  

**Итого OPEX:** ~$3,500–5,000 / месяц  
**Итого за год:** **$42,000–60,000**

---

# 3. Пятый год (масштаб 500k–1M пользователей)

## CAPEX (разовые)
- Полная миграция в распределённую архитектуру: **$20,000–40,000**
- Оптимизация БД, шардирование, аналитика: **$10,000–25,000**
- Развитие рекомендательных моделей / ML-инфраструктуры: **$20,000–50,000**

**Итого CAPEX пятого года:** **$50,000–115,000**

## OPEX (ежемесячные)
- Kubernetes (средний кластер): $3,000–5,000  
- PostgreSQL (8–12 ТБ, репликация): $1,200–2,000  
- Kafka (кластер из 5 брокеров): $1,000–1,800  
- S3 (10–20 ТБ): $500–900  
- Мониторинг/Observability: $300–600  
- Поддержка, SRE, дежурства 24/7: $3,000–5,000  

**Итого OPEX:** ~$10,000–18,000 / месяц  
**Итого за год:** **$120,000–216,000**

---

# Примечание

При переходе на собственные сервера / bare-metal стоимость OPEX может снизиться на **30–60%**,  
но резко возрастут CAPEX:
- покупка оборудования,  
- резервирование,  
- обслуживание дата-центров,  
- содержание SRE-команды,  
- сложность апгрейдов и обновлений.
