# Описание сценариев использования приложения

1. Регистрация и вход пользователя
Цель: получить доступ к сервисам приложения.

 Акторы: Пользователь.
 
 Предусловия: пользователь установил приложение.
 
Основной поток:
- Пользователь выбирает «Регистрация».


- Вводит email/телефон, пароль.


- Получает код подтверждения.


- Входит в приложение.


Результат: создан аккаунт, пользователь попадает в профиль.


2. Создание тренировки

Цель: начать отслеживать тренировку.

 Акторы: Пользователь.
 
Основной поток:
- Пользователь открывает раздел «Тренировка».


- Выбирает тип тренировки.


- Запускает таймер → система начинает сбор данных (пульс, шаги).


- Во время тренировки отображаются показатели.


- Пользователь завершает тренировку.


Результат: тренировка сохранена → отправлена в сервис статистики.

3. Отслеживание состояния организма
   
Цель: получить данные о здоровье.

 Акторы: Пользователь, Сервис интеграции.
 
Основной поток:
- Приложение запрашивает данные с фитнес-часов/браслета.


- Получает метрики: пульс, Co2, сон, калории.


- Отправляет данные в сервис статистики.


- Система формирует рекомендации/отчет.


- Результат: актуальные показатели здоровья сохранены.

4. Участие в тематических группах
   
Цель: общаться с другими пользователями.

 Акторы: Пользователь.
 
Основной поток:
- Пользователь ищет группу по интересам.


- Вступает в неё.


- Открывает чат.


- Отправляет и получает сообщения.


Результат: пользователь участвует в группе.

5. Создание совместного занятия
   
Цель: позвать друзей на совместную тренировку.

 Акторы: Пользователь, Друг.
 
Основной поток:
- Пользователь выбирает «Совместное занятие».


- Назначает дату, время, тип занятия.


- Отправляет приглашение друзьям.


- Друг принимает/отклоняет.


- Занятие отображается у всех участников.



6. Получение уведомлений
   
Цель: оставаться в курсе активности друзей.

 Акторы: Пользователь.
 
Основной поток:
- Сервис уведомлений фиксирует новое событие (друг завершил тренировку).


- Отправляет пуш.


- Пользователь открывает уведомление и переходит в статистику друга.



7. Просмотр личной статистики
   
Цель: следить за прогрессом.

 Акторы: Пользователь.
 
Основной поток:
- Пользователь открывает раздел статистики.


- Просматривает графики: пульс, тренировки, показатели здоровья.


- Система генерирует PDF/внутренний отчет (если нужно).



8. Интеграция с фитнес-приложениями
   
Цель: получать данные из внешних источников.

 Акторы: Пользователь, FitBit / Garmin / Apple Health.
 
Основной поток:
- Пользователь привязывает внешний аккаунт.


- Приложение запрашивает разрешения.


- Получает данные и сохраняет их в сервис трекинга.



9. Просмотр отчёта о здоровье
    
Цель: получить анализ организма.

Основной поток:

- Пользователь открывает «Отчет здоровья».


- Система подгружает свежие метрики.


- Генерирует рекомендации.


- Показывает сводку за неделю/месяц.
  
10. Проверка гипотез (A/B-тестирование)
Цель: оценить эффективность изменений интерфейса или функционала приложения  
Акторы: Пользователь, Сервис аналитики  

Основной поток:
- Система формирует эксперимент с выбранной гипотезой (например, новый дизайн кнопки «Начать тренировку»).
- Пользователи делятся на группы:
   - **Группа A** — контрольная, видит старый интерфейс.
   - **Группа B** — экспериментальная, видит новый интерфейс.
- Пользователи взаимодействуют с приложением.
- Сервис аналитики собирает метрики:
   - количество завершённых тренировок,  
   - кликабельность кнопок,  
   - среднее время на экране,  
   - вовлечённость в групповые занятия.
- Система сравнивает показатели между группами и формирует отчет о гипотезе.
- На основе анализа принимается решение:
   - внедрять изменение,  
   - дорабатывать или  
   - отклонять гипотезу.

Результат: получены данные о влиянии изменений на поведение пользователей; принято решение о внедрении улучшений.
