# Концептуальная архитектура 
Для первой версии продукта применяется модульная архитектура.
Каждый модуль представляет отдельную доменную область (training, communication, tracking, activities, integration), и включает:
- собственную доменную модель,
- слой бизнес-логики (services),
- слой хранения данных.
- Такой подход обеспечивает:
- низкую связность между компонентами,
- возможность независимого масштабирования,
- упрощенный переход к микросервисной архитектуре при росте нагрузки.
